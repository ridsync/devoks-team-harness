# Memory Policy — 메모리 계층 SSOT

이 환경에는 메모리 계층이 여러 개 존재한다. 영속 사실이 여러 곳에 흩어지면 drift(불일치)가 생기므로,
**계층별 역할을 아래로 고정**한다. 결정·선호·영속 사실의 단일 진실원천(SSOT)은 **Claude 파일 메모리** 하나다.

---

## 계층별 역할 (확정)

| 계층 | 위치 | 역할 | 운영 규칙 |
|------|------|------|----------|
| **Claude 파일 메모리** | `~/.claude/projects/.../memory/` (`MEMORY.md` + 개별 `.md`) | **영속 사실·결정·피드백·레퍼런스의 SSOT** | 세션을 넘겨 유지할 결정/선호/프로젝트 사실은 **여기에만** 기록. 글로벌 CLAUDE.md의 memory 컨벤션(frontmatter, type, MEMORY.md 인덱스)을 따른다. |
| **context-mode 세션 메모리** | `ctx_search(sort: "timeline")` | **자동 세션 회상 전용** (resume/compaction 시 직전 결정·에러·프롬프트 복구) | 자동 캡처만 신뢰. **수동 큐레이션 금지.** 회상은 배경 컨텍스트로만 쓰고, 영속시킬 사실은 Claude 파일 메모리로 승격한다. |
| **Serena memory** | `write_memory`/`read_memory` (`.serena/memories/`) | **사용 안 함** (Serena 온보딩 노트가 자동 생성되면 그 용도 한정) | 결정·사실 저장 금지(파일 메모리와 중복·drift 방지). Serena는 심볼 탐색·편집 도구로만 사용. |

> `codebase-memory-mcp`의 ADR(`manage_adr`)은 본 프로젝트에서 cbm을 도입하지 않으므로 해당 없음.

---

## 규칙

1. **영속 결정은 SSOT 하나에만** — 세션 간 유지할 사실/결정/피드백은 Claude 파일 메모리에 기록하고 `MEMORY.md`에 인덱스 한 줄을 남긴다.
2. **회상 ≠ 진실원천** — context-mode 회상 결과는 "당시 사실"의 스냅샷이다. 파일·플래그·심볼을 언급하면 추천 전에 현존 여부를 검증한다.
3. **중복 금지** — 같은 사실을 두 계층에 동시에 적지 않는다. 충돌 시 Claude 파일 메모리가 우선이다.
4. **승격 패턴** — 세션 메모리에서 발견한 영속 가치 있는 사실은 Claude 파일 메모리로 옮겨 적는다(요약 후 인덱싱).
