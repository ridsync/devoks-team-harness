## 0. 적용 원칙

- 이 정책에 **명시적으로 허용되지 않은 것은 금지**로 간주한다(allowlist 우선, No Implicit Fallback).
- 정책 위반 소지가 있는 액션은 실행 전 **차단하고 에스컬레이션**한다(§6). 추측으로 진행하지 않는다.
- 핵심 흐름(구현·검증·딜리버리)은 **Fail Fast**, 비핵심(알림·리포트)은 **graceful degradation**.

---

## 1. 자동화 허용 범위 (scope)

RUN이 승인 없이 수행할 수 있는 것:
- 작업 브랜치 생성/체크아웃, 대상 프로젝트 소스 **코드 편집**(worktree 격리 내).
- 테스트·lint·타입체크·빌드 등 **읽기/검증 명령** 실행.
- 검증 체인(§4) 실행 및 결과에 따른 **자동 수정 루프**(상한 §5).
- Conventional Commits 커밋, **draft PR** 생성 및 본문 작성(추적성·테스트·리스크 요약).
- 원본 이슈 코멘트·Slack 알림 발송(§6, 어댑터 배선 후).

---

## 2. 금지 경로 (deny-list) — 절대 자동 수행 금지

아래는 RUN이 **자동으로 하지 않는다**. 필요하면 차단 후 에스컬레이션(§6):

| 범주 | 금지 액션 |
|---|---|
| **Git 파괴적** | `--force`/`--force-with-lease` push, `git reset --hard`(공유 브랜치), 브랜치·태그 삭제, `main`/기본 브랜치 **직접 push**, `--no-verify` |
| **머지** | PR **auto-merge**, 리뷰 게이트 우회 머지 (D1: draft PR only) |
| **시크릿** | `.env*`·키·토큰·credential 파일 **생성/수정/커밋/출력**, 시크릿 값 로그·리포트 노출 |
| **네트워크 egress** | 샌드박스 allowlist(§3) 밖 아웃바운드 네트워크, 외부로 소스/데이터 업로드 |
| **인프라·데이터** | DB 마이그레이션 실행, 프로덕션 배포, 인프라(IaC·클라우드 리소스) 변경 |
| **파일 삭제** | 소스/설정 파일 **삭제**(rename/이동은 편집 범위 내 허용, 삭제는 에스컬레이션) |
| **컴플라이언스** | 고객 데이터·라이선스가 걸린 변경(§8 자동화 제외 목록) |
| **권한 상승** | 토큰 스코프 확대, 새 permission 요청 자동 승인 |

## 3. 격리 정책 (D2·D6)

- **필수 전제:** 대상 프로젝트에서 RUN은 **네트워크 차단 OS 샌드박스 + worktree 격리** 안에서 실행한다(파일시스템 쓰기는 worktree 내부로 한정). worktree만으로는 불충분하므로 네트워크 차단을 함께 건다(리서치 정론).
- **worktree 병렬:** 독립 태스크는 worktree별 병렬 구현 허용(D3). `devoks-sdlc`(HITL)의 순차 위임 규칙은 이 정책이 건드리지 않는다 — 병렬 해제는 orchestration RUN 한정.
- **네트워크 allowlist 예외(최소 개구부):**
  - 실동작 검증 스테이지(§4, D6)에 한해 **localhost(dev server) + `127.0.0.1:9269`(Chrome DevTools attach)** 만 허용.
  - 패키지 설치 등으로 레지스트리 접근이 필요하면 **사전 승인된 레지스트리 호스트만** allowlist에 추가(RUN 중 임의 확대 금지).
- **롤백:** 편집 전 체크포인트 + git을 병행한다. 게이트 실패·중단 시 worktree 폐기 또는 마지막 성공 체크포인트로 원복.

---

## 4. 검증 게이트 정책 (D2·D6)

- **검증 체인 순서 SSOT = `plugins/devoks-sdlc/skills/feature-workflow-runner/references/post-implementation-checklist.md`** — 이 정책은 순서를 복제하지 않고 참조한다. 현재 확정 순서 요지: 요구사항 → 코드리뷰 → 데이터 흐름 → UI 시각 품질 → 브라우저 실동작 검증 → 테스트 회귀.
- **진행 게이트:** `traces` 커버리지 **0 누락** + `test: required` 태스크 **전부 통과** 여야 딜리버리로 진행.
- **심각도 차단:** 코드리뷰·보안에서 심각도 임계(**Critical/High**, 확정) 발견 시 **draft PR 유지 + 머지 차단 + 에스컬레이션**.
- **브라우저 실동작 검증(D6, 조건부):**
  - dev server 기동 + Chrome(9269 attach) **감지 시** → `verify-acceptance-test`를 **커밋 직전 직렬 게이트**로 자동 실행(§3 allowlist 예외 적용).
  - **미감지 시** → 게이트 강제하지 않고 draft PR에 **"실동작 검증 필요" 플래그** 부착(on-the-loop 위임).
  - Chrome attach는 싱글턴 공유 자원이라 **병렬 불가** → 병합 후 단일 직렬 게이트로 수렴.

## 5. 재시도·비용 한도 (Fail-safe)

| 항목 | 값 |
|---|---|
| 게이트 실패 → 자동 수정 루프 상한 | **2회** (확정) — 초과 시 blocked 처리·에스컬레이션 |
| 태스크 단위 재시도 상한 | **2회** (확정) |
| 동시 병렬 구현 태스크(worktree) 상한 | **4** (확정) — RUN 내 동시 구현 태스크 수. 총 서브에이전트 수는 이 동시성 + 재시도 상한으로 bound |
| RUN당 토큰/비용 상한 | **RUN 시작 시 예산 고정 · 초과 시 중단·에스컬레이션** (원칙 확정, 예산 수치는 호출 시 지정) |
| 동시 실행 RUN 수 / 큐 백프레셔 | 단계 4 소관 — MVP는 **단일 RUN** 전제 |

- 상한 도달은 **조용한 축소 금지** — 무엇이 왜 잘렸는지 RUN 리포트에 명시(No Silent Truncation).

---

## 6. 에스컬레이션 정책 (D5)

**트리거 조건:** 게이트 실패(수정 루프 상한 초과), 불확실성 자기판정(§7), `blocked`(스펙 공백·설계 충돌), deny-list 저촉, 한도 초과.

**채널(이중, 동시):**
- 원본 이슈(Linear) 코멘트 — 사유 · 부분 변경 파일 목록 · 1~2개 해결 제안 · RUN 링크.
- Slack 알림 — 동일 요약 + 이슈/PR 링크.
- **MVP fallback:** 어댑터(Linear/Slack MCP·CLI) 미배선 시 **콘솔 + RUN 종료 리포트**로 대체(단계 2~3에서 배선).

**금지:** 시크릿 값·전체 diff 원문을 에스컬레이션 메시지에 노출하지 않는다(요약·경로·라인 참조만).

---

## 7. 불확실성 자기 게이트

- 복잡도·모호성이 높아 잘못 진행할 위험이 크면, RUN은 **추측 대신 스스로 멈추고 에스컬레이션**한다(리서치: 복잡도가 오를수록 모델이 더 자주 질문 — 이 신호를 게이트로 활용).
- 스펙(FRD/AC)이 결정을 내리기에 불충분하면 `blocked`로 분류하고 §6.

---

## 8. 자동화 제외 목록 (컴플라이언스)

아래에 해당하는 변경은 **RUN 범위에서 제외**하고 항상 사람에게 위임한다(2026-07-23 확정):
- 고객/개인정보(PII) 데이터가 관여하는 변경.
- 라이선스·법무 검토가 필요한 의존성 추가/교체.
- 보안 경계(인증·인가·암호화) 핵심 로직 변경 — draft PR로 만들되 **머지 차단 + 필수 사람 리뷰** 플래그.
- 인프라·DB 마이그레이션 — §2 deny-list와 **이중 명시**(자동 실행 금지 + RUN 범위 제외, 항상 사람 위임).
