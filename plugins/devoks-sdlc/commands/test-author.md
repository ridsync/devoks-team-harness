# 테스트 작성/확장

## Overview

지정된 **파일/모듈/심볼**을 대상으로 자동화 테스트를 작성하거나 기존 테스트를 확장한다.

작성·자기검증은 `test-writer` 에이전트에 위임하고, **리포트만** 돌려받는다.
출력 형식은 `devoks-sdlc:test-author` 스킬을 따른다.

---

## Steps

### 1. 대상 확정

테스트 작성 시작 전에 아래를 확인한다.

1. **대상** — 테스트할 파일/모듈/심볼 경로. 미지정 시 직전 작업·변경 범위 또는 사용자가 가리킨 대상으로 확인한다.
2. **연관 PLAN/FRD(선택)** — EARS Acceptance Criteria와 케이스를 `traces`로 연결해야 하면 경로를 확인한다.

대상이 불명확하면 진행 전에 반드시 한 줄로 재질문한다.

### 2. test-writer 에이전트 호출

대상 확정 후 `test-writer` 에이전트를 아래 파라미터로 호출한다.

| 파라미터 | 값 |
|---------|-----|
| `target` | 1단계에서 확정한 파일/모듈/심볼 경로 |
| `plan` / `frd` | 1단계에서 확인한 경로(있으면) |

> ⚠️ **이 단계는 Pass-through 전용이다.** 메인 에이전트는 subagent 리포트를 요약·압축·재작성·생략해서는 안 된다. 기본 brevity 설정과 무관하게, subagent가 반환한 리포트 전문을 **한 글자도 빠뜨리지 않고 그대로** 사용자에게 출력한다.

**금지:**
- 케이스 표·검증 근거(실행 명령·종료코드·출력 tail) 축약
- 자체 해석이나 추가 코멘트 삽입(리포트 본문 앞뒤에 붙이는 것도 금지)

### 3. 후속 조치 확인

- 리포트에 **발견된 프로덕션 버그**가 있으면(테스트는 수정하지 않고 보고만 함) 수정 진행 여부를 사용자에게 확인한다.
- `status: blocked`면 blocker를 그대로 전달하고 해소 방향을 확인한 뒤 재위임한다.
