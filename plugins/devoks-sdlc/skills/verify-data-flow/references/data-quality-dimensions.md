# 데이터 품질 차원 체크리스트 (Phase 0·4)

DAMA-DMBOK · ISO/IEC 25012 의 데이터 품질 차원을 검증용 체크리스트로 정리한 것이다.
대상 데이터에 **해당하는 차원만 골라서** 적용한다 — 전부 적용할 필요는 없다.

## 사용법

- Phase 0: 검증 대상 인벤토리를 만들 때 각 값에 어떤 차원이 핵심인지 표시.
- Phase 4: 수집된 증거로 차원별 질문에 답하고 통과/실패를 기록.
- 차원별 "통과 기준"은 `<SpecSource>` 에서 끌어와 구체화한다.

---

## 1. 완전성 (Completeness)

필요한 값이 빠짐없이 존재하는가.

- 필수 필드/컬럼에 null/undefined/빈 문자열이 없는가?
- 회차·세트 등 반복 입력에서 누락 회차가 있는가?
- 부분 저장(필드 일부만 갱신)으로 일관성이 깨지지 않았는가?

## 2. 유효성 (Validity)

값이 정해진 타입·범위·형식·enum 을 지키는가.

- 자료형(number/string/boolean)이 맞는가? NaN/Infinity 가 섞이지 않았는가?
- 범위(min/max), `clamp` 가 의도대로 작동했는가?
- 형식(date, email, regex)·enum(상태값)이 허용 집합인가?
- 단위(ms/sec, g/kg, 원/달러)가 일관된가?

## 3. 정확성 (Accuracy)

계산·변환 결과가 기준 공식과 일치하는가.

- `<SpecSource>` 의 공식을 **직접 재현 계산**해 결과가 일치하는가?
- 반올림/내림/유효숫자 정책이 기준대로인가?
- 중앙값/평균 등 통계 산식의 정렬·제외 규칙이 정확한가?
- 이상치(outlier)가 결과를 왜곡하지 않게 처리됐는가?

## 4. 일관성 (Consistency)

같은 사실이 여러 곳에서 어긋나지 않는가.

- **3중 대조**: 적용 의도값 == 영속 저장값 == 재로드 스냅샷.
- 동일 정보가 여러 저장소에 있다면 동기화돼 있는가? (예: in-memory 상태 vs DB)
- 좌/우, A/B 같은 대칭 필드의 정합(예: bothGrindEnabled=true 일 때 좌·우 동기화).

## 5. 무결성 (Integrity)

관계·누적·참조가 깨지지 않는가.

- 참조 무결성: FK 가 가리키는 부모가 존재하는가? 고아 행이 없는가?
- **누적 무결성**: 증분 누적값에서 이중 합산이 없는가? `next == before + delta` 가 성립하는가?
- 트랜잭션 경계가 지켜져, 부분 실패로 중간 상태가 영속되지 않았는가?
- 삭제 시 의존 데이터가 정리(또는 보호)됐는가?

## 6. 유일성 (Uniqueness)

같은 사실이 중복으로 기록되지 않는가.

- 유니크 제약(키/조합)이 지켜지는가?
- 같은 이벤트가 두 번 발화·저장되지 않는가? (idempotency)
- 회차 입력에서 동일 round 가 중복 제출되지 않는가?

## 7. 적시성 (Timeliness)

다음 사용 시점에 갱신값이 반영되는가.

- 저장 직후 다음 추출/요청/계산에서 새 값이 읽히는가? (캐시·메모이션 함정)
- 보정·튜닝 값이 의도한 다음 회차부터 적용되는가, 아니면 한 회차 늦게 반영되는가?
- 비동기 큐/배치 지연이 정합성에 영향을 주는가?

## 정적 무결성 점검 항목 (DB / 파일)

코드 계측 없이 기존 데이터의 무결성을 점검할 때 자주 보는 항목:

- **null 점검** — 필수 컬럼이 null 인 행 수
- **범위/enum 위반** — 정의된 범위·집합 밖 값
- **유일성** — 키/조합의 중복
- **FK 고아** — 자식 → 부모 매칭 실패
- **합계/카운트 reconciliation** — 변환 전후 행수·합계 차이가 의도된 필터를 넘어가는가
- **스키마 일치** — 컬럼·타입·필수 여부가 스펙과 동일한가
- **체크섬** — 파일 단위 무결성(MD5/SHA)
