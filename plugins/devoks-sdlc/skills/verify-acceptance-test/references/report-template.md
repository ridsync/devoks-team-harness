## 작성 팁

- 시나리오 표는 Phase 0에서 만든 표를 그대로 이어받는다 — 새로 설계하지 않는다.
- FAIL은 반드시 증거(스크린샷 파일명, 조회한 실제 값)와 함께 남긴다. "안 되는 것 같다"는 결론이 아니다.
- 검증 환경 갭과 FAIL을 섞지 않는다 — 갭은 "환경이 상태를 재현 못함", FAIL은 "기능이 실제로 틀림".
