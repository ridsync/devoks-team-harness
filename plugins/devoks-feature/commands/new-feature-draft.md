# Function Draft Implement (기능 구현)

## Overview

기능 명세를 받아 프로젝트 패턴에 맞게 기능을 설계·구현한다. UI 구현과 달리 **비즈니스 로직, API 연동, 상태 관리, 데이터 흐름**에 중점을 둔다. **요구 정리 → 설계·승인 → 로컬 브랜치 생성 → 구현 → 마무리** 순서를 따른다.

## 입력 파라미터

실행 시 아래 정보를 제공받는다.

| 파라미터 | 설명 | 예시 |
|----------|------|------|
| **1) 기능 명세** | 구현할 기능의 설명, 범위, 동작 방식 | `목록 CRUD, API 연동, 페이지네이션` |
| **2) 구현 추가정보** | 참고 파일, API 스키마, 제약 사항 등 | `목록 페이지 참고. /api/v1/items 사용` |

추가정보에 다음이 있으면 우선 반영한다.

- **참고 파일**: `@경로`, `~처럼`, `기존 OO 참고`
- **API/엔드포인트**: `API 연동`, `스키마명`, `openapi`
- **상태/저장**: `로컬 저장`, `캐시`, `오프라인`
- **제약**: `인증 필요`, `폼 검증`, `접근성`

## Steps

### 1. 요구사항 정리

- 기능 범위, 입력/출력, 에러 케이스 파악
- 기존 유사 기능 탐색 (`Grep`, `Glob`)
- API 스키마·계약·인증 방식 확인 (있는 경우)

### 2. 설계 및 사용자 확인

다음을 정리해 **승인을 받은 뒤** 코드 생성에 들어간다.

- 영향 받는 모듈·파일·경로
- 데이터 흐름 (API → 상태 → UI)
- 에러 처리·로딩·빈 상태 전략
- 신규 API·스키마·상수 정의

> 사용자 규칙에 "승인 후 코드 생성"이 있으면 이 단계를 생략하지 않는다.

### 3. 로컬 작업 브랜치 생성

구현(코드 작성)에 들어가기 **직전**에, 태스크 목적과 [`devoks-git-commit-msg.md`](devoks-git-commit-msg.md)의 Conventional Commits **타입**에 맞는 로컬 브랜치를 만든다.

- **타입**: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore` 중 작업 성격에 가장 가까운 것을 고른다(상세는 `devoks-git-commit-msg.md`의 Rules).
- **이름 패턴**: `타입/간단-슬러그` — 슬러그는 **kebab-case**·짧게, 이슈·티켓 번호가 입력에 있으면 `타입/번호-슬러그` 형태로 포함해도 된다(예: `feat/128-partner-etc-page`, `fix/mqtt-reconnect-race`).
- **기준 브랜치**: 원격 기본 브랜치(예: `main`) 최신을 기준으로 분기한다(`git fetch` 후 `git checkout -b …` 등 저장소 관례에 맞게).
- **범위**: 브랜치 생성·체크아웃만 수행한다. **커밋·푸시**는 사용자가 명시적으로 요청할 때만 한다(아래 **Rules** 및 프로젝트 규칙).

### 4. 구현

- 비즈니스 로직과 UI 분리 (훅, 유틸, Provider 등)
- 입력 검증·계약 준수
- 로딩·에러·빈 상태 UI 처리
- i18n 키 추가 (`ko.json`, `en.json`)

### 5. 마무리

- `npm run lint`(Bash)로 린트 확인
- 미사용 import 제거
- 구현 요약 및 남은 작업(추후 API, 테스트 등) 안내

## 참조 룰 (Rules)

기능 구현 시 `.claude/rules/` 및 `.claude/refs/` 디렉터리의 규칙 파일들을 참조한다. **이 섹션에 내용을 복사하지 않고 원본 파일을 직접 읽어 적용한다.**

---

## Rules (기능 구현 시 준수)

- **승인 우선**: 구현 계획 후 사용자 승인을 받은 다음 코드 생성
- **패턴 우선**: 기존 코드 스타일·구조를 따르고, 신규 패턴 도입 시 일관성 유지
- **i18n**: 노출 문자열은 `t('key')`, `ko.json`/`en.json` 동시 추가
- **에러 경로**: 모든 비동기·API 호출에 에러 처리 및 사용자 안내
- **Git**: 로컬 작업 브랜치 생성·체크아웃은 Steps에 따른다. **커밋·푸시**는 사용자 명시 요청 시에만 수행한다.

- [ ] i18n 키 추가 (`ko.json`, `en.json`)
