# 시각 diff 체크리스트 템플릿

Figma 스크린샷(`figma_<screen>.png`)과 라이브 렌더(`render_<screen>.png`)를 비교할 때 사용하는 체크리스트. 화면 카테고리별로 점검 항목이 다르다.

---

## 체크리스트 작성 방법

각 항목에 아래 마커를 사용:
- `✅` — 일치 (Figma 와 동일)
- `⚠️ N px off` — 오차 있음 (허용 범위 이내)
- `❌ <설명>` — 불일치 (수정 필요)
- `N/A` — 해당 없음

**px 오차 허용 범위**: 프로젝트마다 다르나 일반적으로 ±4px(device px) 이내를 허용. 중요 요소(CTA 버튼, 타이포 사이즈)는 0px.

## 통과 기준

| 항목 유형 | 허용 오차 |
|----------|----------|
| 레이아웃 (위치/크기) | ±4px (device px) |
| 폰트 크기 | 0px (정확 일치 권장) |
| 폰트 굵기 | 0 (정확 일치 권장) |
| 색상 | 육안으로 동일하거나 토큰 매핑 확인 시 통과 |
| border-radius | ±1px |
| 최소 터치 타깃 | 0px (기준 미달 시 수정 필수) |

> 오차 허용 범위는 프로젝트 디자인 가이드 (`<DesignGuide>`) 에 별도 기준이 있으면 그것을 우선 따른다.

```markdown
| # | 항목 | 결과 | 메모 |
|---|------|------|------|
| 1.1 | 메시지 박스 위치 top | ✅ | |
| 1.2 | 메시지 박스 크기 | ⚠️ 3px off | width 526px → 529px |
| 1.5 | 텍스트 line 1 폰트 크기 | ✅ | |
| 1.8 | 액션 버튼 위치 right | ❌ 12px off | 우측 정렬 안 됨 |

```
